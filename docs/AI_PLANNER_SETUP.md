# AI Planner Setup Guide

## Обзор

AI Planner - это функция интеллектуального планирования проектов, которая использует OpenAI GPT-4 для автоматической декомпозиции проектов на задачи и распределения их между участниками команды.

## Требования

1. **OpenAI API Key** - получите ключ на https://platform.openai.com/api-keys
2. **MySQL Database** - база данных должна быть запущена
3. **Node.js 18+** - для работы Next.js

## Шаги установки

### 1. Добавьте OpenAI API Key в .env

```bash
# Скопируйте .env.example если еще не сделали
cp .env.example .env

# Добавьте ваш OpenAI API ключ
OPENAI_API_KEY=sk-your-actual-api-key-here
```

### 2. Выполните миграцию базы данных

```bash
# Сгенерируйте Prisma Client с новыми моделями
npx prisma generate

# Создайте миграцию для новых таблиц
npx prisma migrate dev --name add_ai_planner

# Или примените миграцию в production
npx prisma migrate deploy
```

### 3. Перезапустите сервер разработки

```bash
npm run dev
```

## Структура базы данных

Были добавлены следующие таблицы:

### AIPlan
Хранит сгенерированные AI планы проектов
- `id` - уникальный идентификатор
- `projectId` - связь с проектом
- `prompt` - исходное описание проекта от пользователя
- `status` - статус плана (draft, approved, rejected)
- `generatedPlan` - JSON с сгенерированными задачами
- `createdBy` - ID создателя плана
- `createdAt`, `updatedAt` - временные метки

### AIPlanMessage
Хранит историю диалога с AI для итеративного уточнения
- `id` - уникальный идентификатор
- `planId` - связь с планом
- `role` - роль сообщения (user, assistant, system)
- `content` - содержимое сообщения
- `createdAt` - время создания

### AIPlanIteration
Хранит все итерации плана для истории изменений
- `id` - уникальный идентификатор
- `planId` - связь с планом
- `iterationNum` - номер итерации
- `prompt` - промпт для этой итерации
- `generatedPlan` - JSON плана для этой итерации
- `createdAt` - время создания

## Использование

### Доступ к AI Planner

1. Откройте страницу проекта
2. Нажмите кнопку "🤖 AI Планировщик"
3. Введите описание проекта
4. Нажмите "Сгенерировать план"
5. Просмотрите и отредактируйте сгенерированные задачи
6. Нажмите "Утвердить и создать задачи"

### Итеративное уточнение

После генерации плана вы можете:
- Задать уточняющие вопросы AI
- Попросить изменить приоритеты
- Перераспределить задачи
- Добавить или удалить задачи

### Права доступа

- **Тимлид** - может создавать и утверждать AI планы
- **Администратор** - полный доступ ко всем AI планам
- **Участник команды** - может просматривать утвержденные планы

## API Endpoints

### POST `/api/teams/[teamId]/projects/[projectId]/ai-plan/generate`
Генерирует новый AI план

**Body:**
```json
{
  "prompt": "Описание проекта",
  "additionalContext": "Дополнительный контекст (опционально)"
}
```

### GET `/api/teams/[teamId]/projects/[projectId]/ai-plan/[planId]`
Получает конкретный AI план

### PATCH `/api/teams/[teamId]/projects/[projectId]/ai-plan/[planId]/refine`
Уточняет существующий план

**Body:**
```json
{
  "refinementPrompt": "Что нужно изменить в плане"
}
```

### POST `/api/teams/[teamId]/projects/[projectId]/ai-plan/[planId]/approve`
Утверждает план и создает задачи в проекте

## Стоимость использования

AI Planner использует GPT-4 Turbo, стоимость примерно:
- Генерация плана: ~$0.01-0.05 за запрос
- Уточнение плана: ~$0.01-0.03 за запрос

Рекомендуется установить лимиты использования в настройках OpenAI.

## Troubleshooting

### Ошибка "OPENAI_API_KEY не настроен"
- Убедитесь, что добавили ключ в `.env` файл
- Перезапустите сервер разработки

### Ошибка "Property 'aIPlan' does not exist"
- Выполните `npx prisma generate`
- Перезапустите TypeScript сервер в VS Code

### Медленная генерация
- GPT-4 может занимать 10-30 секунд
- Это нормально для сложных проектов
- Рассмотрите использование GPT-3.5-turbo для более быстрых ответов

## Безопасность

- API ключ хранится только на сервере
- Никогда не коммитьте `.env` файл в git
- Используйте переменные окружения в production
- Ограничьте доступ к AI Planner только для тимлидов
