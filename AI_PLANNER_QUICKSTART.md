# 🤖 AI Planner - Быстрый старт

## Что было реализовано

Полнофункциональная система интеллектуального планирования проектов с использованием OpenAI GPT-4:

### ✅ Основные возможности

1. **Автоматическая декомпозиция проектов** - AI разбивает описание проекта на конкретные задачи
2. **Умное распределение** - Автоматическое назначение исполнителей с учетом их ролей и навыков
3. **Итеративное уточнение** - Возможность задавать уточняющие вопросы и корректировать план
4. **История изменений** - Сохранение всех итераций плана
5. **Оценка времени** - AI оценивает трудозатраты на каждую задачу
6. **Зависимости задач** - Автоматическое определение связей между задачами

### 📁 Созданные файлы

#### Backend
- `prisma/schema.prisma` - Обновлена схема БД (модели AIPlan, AIPlanMessage, AIPlanIteration)
- `lib/ai-planner.ts` - Утилита для работы с OpenAI API
- `app/api/teams/[teamId]/projects/[projectId]/ai-plan/generate/route.ts` - Генерация плана
- `app/api/teams/[teamId]/projects/[projectId]/ai-plan/[planId]/route.ts` - Получение/удаление плана
- `app/api/teams/[teamId]/projects/[projectId]/ai-plan/[planId]/refine/route.ts` - Уточнение плана
- `app/api/teams/[teamId]/projects/[projectId]/ai-plan/[planId]/approve/route.ts` - Утверждение и создание задач

#### Frontend
- `app/dashboard/teams/[teamId]/projects/[projectId]/ai-planner/page.tsx` - Страница AI-планировщика

#### Документация
- `docs/AI_PLANNER_SETUP.md` - Подробная документация
- `.env.example` - Обновлен с OPENAI_API_KEY

## 🚀 Запуск

### Шаг 1: Добавьте OpenAI API Key

```bash
# Откройте .env файл и добавьте:
OPENAI_API_KEY=sk-your-actual-api-key-here
```

Получить ключ: https://platform.openai.com/api-keys

### Шаг 2: Выполните миграцию базы данных

```bash
# Сгенерируйте Prisma Client
npx prisma generate

# Создайте миграцию
npx prisma migrate dev --name add_ai_planner

# Или примените существующую миграцию
npx prisma migrate deploy
```

### Шаг 3: Перезапустите сервер

```bash
npm run dev
```

## 📖 Как использовать

### 1. Откройте проект
Перейдите на страницу любого проекта в системе

### 2. Нажмите "🤖 AI Планировщик"
Кнопка доступна только для тимлидов и администраторов

### 3. Опишите проект
Введите подробное описание того, что нужно реализовать:

```
Пример:
Создать веб-приложение для управления задачами с возможностью:
- Создания проектов и команд
- Назначения исполнителей
- Отслеживания прогресса
- Генерации отчетов
- Аутентификации и ролей пользователей

Технологии: Next.js, TypeScript, Prisma, MySQL
Срок: 2 месяца
Команда: 2 frontend, 1 backend разработчик
```

### 4. Сгенерируйте план
AI проанализирует описание и создаст структурированный план задач

### 5. Уточните при необходимости
Можете задать уточняющие вопросы:
- "Добавь задачу по написанию документации"
- "Увеличь приоритет задачи #3"
- "Перераспредели задачи более равномерно"

### 6. Утвердите план
После утверждения все задачи будут автоматически созданы в проекте

## 🔧 Настройка

### Изменить модель AI
В файле `lib/ai-planner.ts` измените модель:

```typescript
model: "gpt-4-turbo-preview", // или "gpt-3.5-turbo" для экономии
```

### Настроить промпт
Отредактируйте `SYSTEM_PROMPT` в `lib/ai-planner.ts` для изменения поведения AI

### Лимиты и стоимость
- GPT-4 Turbo: ~$0.01-0.05 за запрос
- GPT-3.5 Turbo: ~$0.001-0.01 за запрос

Рекомендуется установить лимиты в настройках OpenAI

## 🐛 Troubleshooting

### "OPENAI_API_KEY не настроен"
→ Добавьте ключ в `.env` и перезапустите сервер

### "Property 'aIPlan' does not exist"
→ Выполните `npx prisma generate`

### Медленная генерация
→ Это нормально для GPT-4 (10-30 секунд)
→ Используйте GPT-3.5-turbo для более быстрых ответов

### Ошибки миграции
→ Убедитесь, что база данных запущена
→ Проверьте DATABASE_URL в .env
